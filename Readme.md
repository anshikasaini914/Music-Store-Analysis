🎵 Music Store Data Analysis using PostgreSQL
📌 Overview

This project analyzes a Music Store relational database using PostgreSQL to extract meaningful business insights.
The dataset contains information about customers,employee, playlist, invoices, artists, albums, tracks, and genres.

Using SQL queries, the project answers business questions related to sales performance, customer behavior, and music preferences.

The goal of this project is to demonstrate SQL skills, relational database analysis, and data-driven decision making.

⭐ Project Highlights

1. Analyzed a multi-table relational database using PostgreSQL.

2. Solved 15+ real-world business questions using SQL.

3. Used joins, aggregations, subqueries, and filtering to extract insights.

4. Identified top customers, revenue-generating cities, and popular music genres.

5. Performed customer spending analysis across artists and countries.

6. Organized the project with clean SQL scripts, dataset, and documentation.

🛠 Tools & Technologies

PostgreSQL – Database management

SQL – Data querying and analysis

CSV Dataset – Music store dataset


🔎 Analysis Steps

1. Created tables in PostgreSQL using SQL schema.

2. Imported dataset into the database.

3. Explored relationships between tables.

4. Wrote SQL queries to answer business questions.

5. Extracted insights from the query results.
